Working through this Ubuntu lab gave me practical insight into the trade-offs between graphical and command-line interfaces. The GUI (File Manager, Firefox, Ubuntu Software Centre) felt intuitive and low-friction for everyday tasks like browsing the web or checking network settings. However, the CLI proved far more powerful and efficient for system-level operations—editing /etc/hosts, compiling C code with gcc, or running apt update to refresh package lists. The CLI's speed and scriptability are unmatched once you memorise key commands.

Regarding software installation, each method has clear pros and cons. Repository installs (apt or Ubuntu Software Centre) are the safest and most convenient—automatic updates, dependency resolution, and Ubuntu-tested security. SaaS (Google Docs, Office 365) requires no local installation and works across devices, but depends entirely on internet connectivity. Manual binary downloads (.deb files) offer access to software not in repositories (e.g., Chrome), but you must manually check for updates and trust the source. Compiling from source gives maximum control and optimisation, but it's time-consuming and lacks automated updates.

As a user, I prefer repository installations for their convenience and security. As a future software developer, I value understanding all four methods. Knowing how to compile from source and manually manage binaries is essential for working with niche tools or proprietary environments. The CLI, while having a steeper learning curve, is non-negotiable for serious development work. This lab reinforced that both GUI and CLI have their place—the best approach depends entirely on the task at hand.
